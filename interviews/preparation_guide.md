## 2. Face-to-Face Interview

## Technical Questions

### Explain your recent project

In my recent project, I worked on a backend system for managing business operations such as user data, transactions, and reporting.

I was responsible for designing and implementing RESTful APIs, structuring the database schema, and ensuring proper validation and error handling.

I also organized the codebase into modular components to improve maintainability and make future scaling easier.

---

### How do you design an API?

I start by understanding the business requirements and identifying core entities and their relationships.

Then I design RESTful endpoints using proper HTTP methods and consistent naming conventions.  
I also focus on input validation, structured error handling, and standardized response formats.

If needed, I consider scalability aspects such as pagination, filtering, and versioning.

---

### Monolith vs Microservices

A monolithic architecture is a single unified system where all components are tightly coupled.

Microservices architecture breaks the system into smaller, independent services that communicate via APIs.

While microservices offer better scalability and flexibility, they also introduce complexity in deployment, communication, and monitoring.

In my experience, I’ve worked with modular architectures that follow some microservices principles, even if not fully distributed.

---

### Error handling

I handle errors by validating inputs early, returning meaningful and consistent error responses, and logging issues for monitoring and debugging.

I also try to anticipate edge cases to prevent unexpected failures in production.

---

### Database experience

I have experience working with relational databases like MySQL, including schema design, query optimization, indexing, and handling relationships between tables.

I also ensure data consistency and performance when designing queries.

---

## Behavioral Questions

### How do you handle deadlines?

I break tasks into smaller parts, prioritize based on impact, and focus on delivering core functionality first.

I also communicate early if there are risks or blockers to ensure expectations are aligned.

---

### What if you don’t understand a task?

I try to analyze the problem first and gather as much context as possible.

If something is still unclear, I ask targeted questions early to avoid misunderstandings and ensure I’m aligned with expectations.

---

### Teamwork

I’m comfortable working in a team environment. I value clear communication, constructive feedback, and aligning on technical decisions to achieve better results.

---

## 3. Exercise Assessment

### What they expect (IMPORTANT)

- Clean and readable code
- Clear structure (layered / modular)
- Proper naming conventions
- Error handling
- Basic best practices

---

### Your winning approach

- Don’t rush → write clean
- Keep functions small and clear
- Use consistent API response format
- Handle edge cases (empty, invalid, etc.)

---

## Key Strategy (Advanced Level)

### Positioning yourself

- Not junior → but not fake senior
- Show **structured thinking**
- Show **engineering mindset**, not just coding

---

### Strong phrases to use

- “I try to keep the code modular and maintainable”
- “I consider scalability and future changes”
- “I focus on clear and consistent API design”
- “I aim to write code that is easy for others to understand”

---

## Safety Answer (VERY IMPORTANT)

If you don’t know something:

That’s a good question. Based on my understanding, I would approach it by...

→ Then explain your thinking

---

## Final Notes

- Speak clearly, not fast
- Keep answers structured
- Don’t overcomplicate
- Show how you think

---

## Mindset

They are not only hiring your skill.  
They are hiring:

- How you think
- How you communicate
- How reliable you are

→ Make them feel: “This person is safe to work with.”

# Versi 2


Kamu jangan anggap ini cuma “interview bahasa Inggris”—itu justru jebakan yang bikin grogi. Yang dinilai sebenarnya tetap **skill + cara berpikir + komunikasi**. Bahasa Inggris cuma alatnya.

Aku bantu breakdown biar kamu bisa latihan dengan fokus dan realistis.

---

## 1. Yang kemungkinan akan diuji

Dari posisi **Golang Developer**, biasanya mereka cek:

### a. Screening Interview (awal)

Fokus:

* Perkenalan diri (background kamu)
* Pengalaman (project, kerja, freelance)
* Basic English conversation

Contoh:

* “Tell me about yourself”
* “What projects have you worked on?”
* “Why Golang?”

👉 Ini bukan grammar test. Mereka cuma lihat: **kamu bisa nyampein ide atau tidak**

---

### b. Technical Interview

Fokus:

* Golang fundamentals
* API / backend
* mungkin sedikit system design

Contoh:

* Goroutine vs thread
* Channel di Go
* REST API design
* Database (PostgreSQL/MySQL)
* Error handling di Go

---

### c. Assessment (kalau ada)

Biasanya:

* bikin API sederhana
* atau debug code

---

## 2. Masalah utama kamu (jujur aja)

Bukan di skill coding.

👉 Tapi di: **ngomong Inggris sambil mikir teknis**

Ini memang berat kalau belum biasa.

---

## 3. Cara latihan paling efektif (bukan teori doang)

### 🔥 Step 1 — Siapkan jawaban template (WAJIB)

Latih ini dulu, jangan random:

#### 1. Perkenalan diri

```
Hi, my name is Jehan.
I am a software developer with a focus on backend development, especially using Golang and Laravel.

I have experience building web applications, APIs, and some custom systems for clients.

Recently, I’ve been focusing more on improving my backend architecture and writing clean, maintainable code.
```

---

#### 2. Cerita project (ini penting banget)

```
One of my recent projects was building a sales application.

In this project, I was responsible for designing the backend API, managing the database, and implementing business logic.

I used Laravel for fast development, but I’m currently transitioning to Golang for better performance and scalability.
```

---

#### 3. Kenapa Golang

```
I like Golang because it's simple, fast, and great for building scalable backend systems.

I also like its concurrency model using goroutines, which makes it efficient for handling multiple requests.
```

---

👉 Hafalin + pahami, jangan dibaca

---

### 🔥 Step 2 — Latihan ngomong (ini kunci)

Jangan cuma baca.

Lakukan ini:

* ngomong sendiri (keras)
* rekam pakai HP
* ulang 5–10x

Kalau perlu:

* aku bisa jadi interviewer kamu (practice mock interview)

---

### 🔥 Step 3 — Latihan jawaban teknis pakai English simple

Contoh:

❌ Jangan:

> "Concurrency is a complex paradigm..."

✅ Tapi:

```
In Golang, goroutines are lightweight threads.
They allow us to run multiple functions at the same time.

Channels are used to communicate between goroutines safely.
```

👉 simple English > grammar perfect

---

## 4. Mindset penting (ini sering bikin gagal)

### ❌ Salah:

* takut grammar salah
* mikir terlalu lama
* pengen terdengar “pintar”

### ✅ Benar:

* jawab simple
* kalau bingung, bilang:

  ```
  Let me think for a moment
  ```
* kalau ga ngerti:

  ```
  Sorry, could you repeat the question?
  ```
